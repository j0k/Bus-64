# 🔐 Online Crypter

**v1.0.0** · [English version →](README.md)

Стойкое шифрование текста прямо в браузере. Вводишь текст, выбираешь алгоритм и получаешь результат в **base64** — расшифровать можно тем же паролём. Без сервера, без слежки, **без внешних зависимостей**. Один самодостаточный файл `index.html`.

**Онлайн:** https://j0k.github.io/online2b64/

![Превью Online Crypter](preview.png)

## Возможности

- **Три аутентифицированных алгоритма** (все AES-256):
  - **AES-256-GCM** — аутентифицированное шифрование (AEAD), целостность встроена. *Рекомендуется.*
  - **AES-256-CBC + HMAC-SHA256** — Encrypt-then-MAC.
  - **AES-256-CTR + HMAC-SHA256** — потоковый режим с отдельным HMAC.
- **По паролю** — ключ выводится из ключевой фразы через **PBKDF2-SHA256** (210 000 итераций, новая случайная соль 16 байт на каждое шифрование).
- **Вывод в base64** — соль, IV и id алгоритма упакованы в результат, поэтому для расшифровки нужен только пароль.
- **Проверка целостности** — неверный пароль или подменённые данные отклоняются *до* расшифровки (сравнение за постоянное время), а не превращаются в мусор.
- **Двуязычный интерфейс** — переключатель EN / RU, выбор запоминается в `localStorage`.
- **Работает офлайн** и полностью локально — на сервер ничего не уходит.
- Светлая / тёмная тема по системным настройкам.

## Как пользоваться

1. Открой [страницу](https://j0k.github.io/online2b64/) (или `index.html` локально по HTTPS).
2. Выбери алгоритм и введи **пароль**.
3. Введи текст и нажми **🔒 Зашифровать** → скопируй результат в base64.
4. Обратно: вставь base64, введи **тот же пароль**, нажми **🔓 Расшифровать**.

> ⚠️ Восстановления пароля нет. Забыл пароль — данные расшифровать невозможно.

## Как это работает

Ключ выводится из пароля через PBKDF2-SHA256. Шифрование использует встроенный в браузер [Web Crypto API](https://developer.mozilla.org/docs/Web/API/Web_Crypto_API) (`crypto.subtle`).

Структура байтов результата, затем кодируется в base64:

```
[ версия(1) | алгоритм(1) | итерации(4, BE) | соль(16) | IV | шифртекст ] (+ HMAC(32) для CBC/CTR)
```

Для AES-GCM тег аутентификации входит в шифртекст. Для CBC/CTR добавляется HMAC-SHA256 по всему сообщению (Encrypt-then-MAC).

## О безопасности

- **AES-256 уже более чем достаточно** — «AES-512» не существует; AES стандартизирован только для ключей 128/192/256 бит. Реальная стойкость зависит от **пароля**, а не от числа бит ключа.
- Клиентское шифрование защищает данные при хранении и передаче через недоверенные каналы; оно **не** защищает от скомпрометированного устройства или слабого пароля.
- Требуется **HTTPS** (Web Crypto недоступен по обычному `http://` и `file://` в части браузеров). GitHub Pages отдаёт HTTPS.
